a) What are local and global variables?

+

Local Variable: A variable declared inside a function and is accessible only within that function's scope. It's created when the function is called and destroyed when the function returns.

def my_function():
  x = 10  # x is a local variable
  print("Inside function, x =", x)

my_function()
# print(x) # This would cause an error, as x is not defined globally

In the example above, x is only known inside my_function(). Trying to access it outside will result in a NameError.

Global Variable: A variable declared outside of any function, typically at the top level of a script. It can be accessed from any part of the code, both inside and outside functions (though modification inside a function requires the global keyword).

y = 20  # y is a global variable

def another_function():
  print("Inside function, y =", y) # Accessing global y

another_function()
print("Outside function, y =", y)

Here, y is defined globally and can be read by another_function(). To modify a global variable inside a function, you'd use global y.

g) Write the use of range() function.

+

The range() function is a built-in Python function used to generate an immutable sequence of numbers. It's commonly used with for loops to iterate a specific number of times or over a sequence of indices.

Syntax:

  • range(stop): Generates numbers from 0 up to (but not including) stop. 
    for i in range(3): # stop = 3
    print(i) # Output: 0, 1, 2
  • range(start, stop): Generates numbers from start up to (but not including) stop. 
    for i in range(1, 4): # start = 1, stop = 4
    print(i) # Output: 1, 2, 3
  • range(start, stop, step): Generates numbers from start up to (but not including) stop, incrementing by step. 
    for i in range(1, 6, 2): # start = 1, stop = 6, step = 2
    print(i) # Output: 1, 3, 5

Key uses of range():

  • Controlling loop iterations: Execute a block of code a fixed number of times.
  • Generating indices: To access elements of a list or other sequences by their index.
  • Creating numerical sequences for various calculations.
It's memory efficient because it doesn't store all the numbers in memory at once; it generates them on demand.

j) What is tuple? Create a tuple and print it in reverse order.

+

A tuple in Python is an ordered and immutable collection of items. This means once a tuple is created, you cannot change, add, or remove its elements.

  • Tuples can contain items of different data types.
  • They are defined by enclosing comma-separated values within parentheses (). Parentheses are optional if the context is clear.
  • Often used to store related pieces of information that should not be modified, like coordinates or records.

# Creating a tuple
my_tuple = (1, 2, "Python", 3.14, True)

The slicing notation [::-1] creates a reversed copy of the tuple. Since tuples are immutable, this operation doesn't change the original my_tuple; it creates a new tuple that is reversed.

a) Write a function to input a number and check if it is prime number or not.

+

A prime number is a natural number greater than 1 that has no positive divisors other than 1 and itself.

def is_prime(n):
    # Step 1: Handle base cases
    if n <= 1:
        return False  # Numbers less than or equal to 1 are not prime

    # Step 2: Iterate from 2 up to the square root of n
    # We only need to check divisibility up to sqrt(n) because if n has a divisor larger than sqrt(n),
    # it must also have a divisor smaller than sqrt(n).
    for i in range(2, int(n**0.5) + 1):
        # Step 3: Check for divisibility
        if n % i == 0:
            return False  # If n is divisible by any number other than 1 and itself, it's not prime

    # Step 4: If no divisors were found, the number is prime
    return True

# Get input from the user
try:
    num = int(input("Enter a number: "))

    # Check and print the result
    if is_prime(num):
        print(f"{num} is a Prime number")
    else:
        print(f"{num} is Not a Prime number")
except ValueError:
    print("Invalid input. Please enter an integer.")

Line-by-line explanation of is_prime(n):

  • if n <= 1: return False: Prime numbers are greater than 1. So, 0, 1, and negative numbers are not prime.
  • for i in range(2, int(n**0.5) + 1):: This loop iterates through potential divisors.
    • It starts from 2 because 1 is a divisor of all numbers.
    • It goes up to the square root of n (int(n**0.5) + 1). This is an optimization: if n has a divisor larger than its square root, it must also have a divisor smaller than its square root. So, we only need to check up to the square root. We add 1 because range excludes the stop value.
  • if n % i == 0: return False: If n is perfectly divisible by i (meaning i is a divisor other than 1 and n itself), then n is not prime, and the function immediately returns False.
  • return True: If the loop completes without finding any divisors, it means n is only divisible by 1 and itself, so it's a prime number.

b) Write a python program to find the roots of a quadratic equation.

+

A quadratic equation is an equation of the form: ax² + bx + c = 0, where a, b, and c are coefficients and a ≠ 0.

The roots of a quadratic equation can be found using the quadratic formula:

x = (-b ± √(b² - 4ac)) / 2a

The term d = b² - 4ac is called the discriminant. The nature of the roots depends on the value of d:

  • If d > 0, the roots are real and distinct.
  • If d = 0, the roots are real and equal.
  • If d < 0, the roots are complex and distinct (conjugate pairs).
import cmath # Import cmath module for handling complex square roots

def find_quadratic_roots(a, b, c):
    """
    Calculates the roots of a quadratic equation ax^2 + bx + c = 0.
    Args:
        a (float/int): Coefficient of x^2 (must not be zero).
        b (float/int): Coefficient of x.
        c (float/int): Constant term.
    Returns:
        tuple: A tuple containing the two roots (can be complex).
               Returns None if 'a' is zero.
    """
    if a == 0:
        print("Coefficient 'a' cannot be zero for a quadratic equation.")
        return None

    # Calculate the discriminant
    discriminant = (b**2) - (4 * a * c)

    # Find the roots using the cmath.sqrt() for complex number support
    root1 = (-b - cmath.sqrt(discriminant)) / (2 * a)
    root2 = (-b + cmath.sqrt(discriminant)) / (2 * a)

    return root1, root2

# Example usage: ax^2 + bx + c = 0
# Example 1: Two distinct real roots (x^2 - 5x + 6 = 0 => roots are 2, 3)
a1, b1, c1 = 1, -5, 6
roots1 = find_quadratic_roots(a1, b1, c1)
print(f"For equation {a1}x^2 + {b1}x + {c1} = 0, Roots are: {roots1}")
Output:For equation 1x^2 + -5x + 6 = 0, Roots are: ((2+0j), (3+0j))

# Example 2: Two equal real roots (x^2 - 4x + 4 = 0 => roots are 2, 2)
a2, b2, c2 = 1, -4, 4
roots2 = find_quadratic_roots(a2, b2, c2)
print(f"For equation {a2}x^2 + {b2}x + {c2} = 0, Roots are: {roots2}")
Output:For equation 1x^2 + -4x + 4 = 0, Roots are: ((2+0j), (2+0j))

# Example 3: Two complex roots (x^2 + x + 1 = 0)
a3, b3, c3 = 1, 1, 1
roots3 = find_quadratic_roots(a3, b3, c3)
print(f"For equation {a3}x^2 + {b3}x + {c3} = 0, Roots are: {roots3}")
Output:For equation 1x^2 + 1x + 1 = 0, Roots are: ((-0.5-0.8660254037844386j), (-0.5+0.8660254037844386j))

# Get input from user
print("\nEnter coefficients for your quadratic equation (ax^2 + bx + c = 0):")
try:
    coeff_a = float(input("Enter coefficient a: "))
    coeff_b = float(input("Enter coefficient b: "))
    coeff_c = float(input("Enter coefficient c: "))
    user_roots = find_quadratic_roots(coeff_a, coeff_b, coeff_c)
    if user_roots:
        print(f"The roots are: {user_roots[0]} and {user_roots[1]}")
except ValueError:
    print("Invalid input. Please enter numeric coefficients.")

Explanation:

  • import cmath: The cmath module (complex math) is imported because the discriminant (b² - 4ac) can be negative, resulting in complex roots. cmath.sqrt() can handle square roots of negative numbers.
  • if a == 0:: Checks if the equation is truly quadratic. If 'a' is 0, it's a linear equation.
  • discriminant = (b**2) - (4 * a * c): Calculates the discriminant.
  • The two roots root1 and root2 are calculated using the standard quadratic formula.
  • The function returns a tuple containing the two roots. Even if the roots are real, cmath will represent them as complex numbers with an imaginary part of 0 (e.g., (2+0j)).

e) Create a function to find the sum of the given series: 1 + 3 + 5 + 7 + ... + n (n is input)

+

The given series is a sum of odd numbers up to n (if n itself is odd and included) or up to the largest odd number less than or equal to n.

Method 1: Using a loop with a step of 2

def odd_series_sum_loop(limit_n):
    """
    Calculates the sum of the odd series 1 + 3 + 5 + ... up to n.
    Args:
        limit_n (int): The upper limit of the series.
    Returns:
        int: The sum of the odd numbers in the series.
    """
    total_sum = 0
    # Iterate through odd numbers starting from 1 up to limit_n (inclusive)
    # The step is 2 to only consider odd numbers.
    for i in range(1, limit_n + 1, 2):
        total_sum += i # Add the current odd number to the total
    return total_sum

# Get input from the user for n
try:
    n_value = int(input("Enter the value of n for the series (1+3+5+...+n): "))
    if n_value < 1:
        print("Please enter a positive integer for n.")
    else:
        result_sum = odd_series_sum_loop(n_value)
        print(f"The sum of the odd series up to {n_value} is: {result_sum}")
except ValueError:
    print("Invalid input. Please enter an integer.")

# Example Test Cases:
# print(f"Sum for n=7: {odd_series_sum_loop(7)}")  # 1+3+5+7 = 16
# print(f"Sum for n=8: {odd_series_sum_loop(8)}")  # 1+3+5+7 = 16 (8 is even, so 7 is the last odd number)
# print(f"Sum for n=1: {odd_series_sum_loop(1)}")  # 1 = 1

Line-by-line explanation of odd_series_sum_loop(limit_n):

  • total_sum = 0: Initializes a variable to store the sum.
  • for i in range(1, limit_n + 1, 2)::
    • range() generates a sequence of numbers.
    • 1: Starts the sequence from 1 (the first odd number).
    • limit_n + 1: Goes up to limit_n. We add 1 because range excludes the stop value, ensuring limit_n is considered if it's odd.
    • 2: The step value. This makes the loop jump by 2 in each iteration (1, 3, 5, ...), effectively picking only odd numbers.
  • total_sum += i: In each iteration, the current odd number i is added to total_sum.
  • return total_sum: After the loop finishes, the function returns the calculated sum.

Method 2: Mathematical Formula (More Efficient)

The sum of the first k odd numbers is k². We first need to find how many odd numbers (k) are there up to n.

If n is the last odd number in the series, then n = 2k - 1. So, k = (n + 1) / 2.

def odd_series_sum_formula(limit_n):
    """Calculates sum using mathematical formula."""
    if limit_n < 1:
        return 0
    # Find the largest odd number less than or equal to limit_n
    if limit_n % 2 == 0:
        last_odd = limit_n - 1 # If n is even, the last odd in series is n-1
    else:
        last_odd = limit_n    # If n is odd, n is the last odd in series

    if last_odd < 1: # Handles case where n=0 after adjustment
        return 0

    # Number of odd terms (k) in the series up to last_odd
    # For a series 1, 3, 5, ..., m (where m is last_odd), m = 2k - 1 => k = (m+1)/2
    k = (last_odd + 1) // 2 # Use integer division
    return k ** 2 # Sum is k squared
